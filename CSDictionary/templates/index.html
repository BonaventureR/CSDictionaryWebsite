CS Dictionary


What is CS Dictionary?

CS Dictionary uses MicroStrategy's embedded analytics system to create a dictionary for technological jargon and displays it to the user in the form of a hyper-intelligence card. This is useful for people who are trying to learn or have forgotten information related to technology. By showing definitions in the terms of hyper-intelligence cards, it makes it easier for the user to grasp the concepts of harder topics in a webpage without navigating away.




Motivation

In an world where technological evolution is exponential, it is imperative that professionals keep up. There are always new things to learn, remember, and research. CS Dictionary was created with the goal of simplifying that process in mind. Having to constantly Google new and forgotten keywords can be time consuming and CS Dictionary seeks to fix that problem for both technical and non-technical people alike.

Methodology
  • filter_list1. Data CollectionPython
    When beginning our project, it was clear that the natural place to start was with data collection. We needed to find some sort of list or database of technical jargon. Surprisingly, after a long Google search we were unable to find such a collection that fit our needs. This motivated us even more as the need for a tool like CS Dictionary became even more clear. We decided that we would have to generate our own database of words through some form of web scraping. We turned to possibly the most comprehensive resource in the world, Wikipedia. Using Wikipedia's API, we created a script that would gather all our key words using a graph traversal algorithm similar to depth first. One of the final versions of this script can be found here.
  • domain2. Database PopulationAWS and mySQL
    The next step was to take the script that pulled from Wikipedia and make it populate our database. First, we created an AWS RDS (relational database service) using mySQL. Then, after many iterations we arrived at this. We scrapped the key computer science terms from Wikipedia for about an hour and a half and populated our database at the same time. Using our script we were able to gather words, definitions, links and synonyms all at the same time.
  • assessment3. Linking With MicrostrategyMicroStrategy HyperCards
    We created a Microstrategy environment and linked it to our AWS database. From there, we used the data to create HyperCards. This process was fairly simple once we got our database connected. However, due to the nature of chrome apps, the project had to be limited in terms of scope. In the future we may explore implementing something without limitations like this.
